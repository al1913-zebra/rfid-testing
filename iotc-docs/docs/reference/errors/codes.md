> 📕 **REFERENCE** · **Audience:** API Consumer · **Use:** look up codes appearing in command responses

Twenty-nine response codes (0–28). Code `0` is success; codes `1`–`28` indicate specific conditions. Codes are command-scoped — each command's schema lists the subset it can return. The table below is the canonical superset, taken verbatim from `error_codes.json` in the API schema reference.

## How to read the table

- **Code** — the integer in `response.code`.
- **`iot_status_code`** — the internal status-code enum name (useful when correlating with firmware logs or support tickets).
- **Description** — the string in `response.description`, as returned by the device.
- **Cause** — what triggers this code.
- **Recommended action** — what to do as the integrator.
- **Applies to** — the commands that can return this code. `*` means all commands.

## Notes on `cloud_connect`

`cloud_connect` is not exposed as a top-level MQTT command in the public API surface; it is a device-internal operation invoked during bootstrap. Codes 6 (region not configured) and 7 (interface not available) surface to integrators as failures during 123RFID Desktop provisioning rather than as MQTT responses. Code 7 can additionally appear in a [`get_wifi`](https://aa5123.github.io/RFID-40-90-handled-reader-api-reference-documentatiion/#op-get-wifi) response when the Wi-Fi interface is disabled (per that command's API-reference example), even though its documented range is otherwise 0 and 3.

## Notes on [`reboot`](https://aa5123.github.io/RFID-40-90-handled-reader-api-reference-documentatiion/#op-reboot) code 1

The [`reboot`](https://aa5123.github.io/RFID-40-90-handled-reader-api-reference-documentatiion/#op-reboot) API reference example shows a response with `code: 1` ("Command payload is accepted"), but the canonical schema and error table define only `0` (Success) and `5` (Inventory in progress) for [`reboot`](https://aa5123.github.io/RFID-40-90-handled-reader-api-reference-documentatiion/#op-reboot). Treat `0` and `1` as success-equivalents and `5` as the only documented failure when handling [`reboot`](https://aa5123.github.io/RFID-40-90-handled-reader-api-reference-documentatiion/#op-reboot) responses. See [Error Response Format](/reference/errors/format) for the full discussion.
