> 📕 **REFERENCE** · **Audience:** API consumer · **Use:** read the reader's wired Ethernet interface configuration and status

## Overview

`get_eth` retrieves the reader's **wired Ethernet interface configuration and status** — interface state, live link status and speed, and the DHCP-assigned IPv4 addressing. It is a read-only query on the device-management (MGMT) endpoint, useful for verifying that the reader's wired uplink is up and reaching back-end services.

The response shape depends on interface state: when Ethernet is **enabled** the device returns the full `interfaceDetails` (hostname, MAC, link, IPv4); when it is **disabled** it returns only `interfaceName` and `status` (still a `code 0` success). Note that, per the command's own contract, **802.1X security, static IP configuration, and IPv6 are not available in the current API version** (see the notes below).

## When to Use

Use `get_eth` when you need to **verify the reader's wired network state** — during commissioning, provisioning verification, or troubleshooting a wired uplink. Because it is read-only and idempotent, it is safe to poll.

- **Confirm connectivity:** check `interfaceDetails.status` (enabled?), `linkStatus.status` (cable connected?), and `linkStatus.linkSpeed` (negotiated speed).
- **Validate addressing:** check `ipv4Configuration.ipAddress`, `gateway`, and `enableDhcp` to confirm the reader has a working IPv4 route to the broker / back-end.
- **Troubleshoot wired issues:** an unexpected `linkSpeed`, a missing IP, or a `Disabled` interface each point to a specific cable/switch/config problem.

## Key Fields

- **`requestId`** (request) — your correlation token; make it unique per request. The device echoes it in the response so you can pair reply to request.
- **`ethConfig.interfaceDetails.status`** (response) — `Enabled` or `Disabled`. When `Disabled`, no other interface details are returned.
- **`ethConfig.interfaceDetails.linkStatus`** (response) — `status` (`Connected` / `Disconnected`) and `linkSpeed`; the live carrier state.
- **`ethConfig.interfaceDetails.ipv4Configuration`** (response) — the DHCP-assigned IPv4 addressing (reported read-only).
- **`response.code`** (response) — read this to determine success vs failure (see [Response Codes](#response-codes)).

## Response payload

The response envelope is `command`, `requestId`, `apiVersion`, `response`, and an `ethConfig` object. On the **Schema** tab, expand `ethConfig` → `interfaceDetails` (and its `linkStatus`, `ipv4Configuration`, `security` sub-objects) to see every field — its type, whether it is required, its enum/units, an example value, and a description; the **Example** tab shows the full (enabled-interface) JSON. Within `interfaceDetails`, only `interfaceName` and `status` are required; the rest are returned when the interface is **enabled**. The `security` block is reserved (not available in the current API version), and `ipv4Configuration` is reported read-only.

## Response Codes

`get_eth` returns `response.code` per `response.yaml` and `error_codes.json`. The codes applicable to this command are below; for the full table see the [Error codes reference](/reference/errors/codes) and [Error format](/reference/errors/format).

| Code | Description |
| --- | --- |
| `0` | Success — Ethernet configuration retrieved and returned. A **disabled** interface is also a `0` success (minimal payload), not an error. |
| `3` | Not able to retrieve information (`IOT_ERROR_INFO_NOT_AVAILABLE`) — the device could not gather the interface details at this time. Retry after a short delay; if it persists, reboot the device. |
| `101` | Default error — generic catch-all. |

## Notes & Caveats

:::warning
**Not available in the current API version.** Per this command's contract, **802.1X security, static IP configuration, and IPv6 are not supported in the current API version.** The response schema models a `security` / `EAP802_1X` block and a full `ipv4Configuration`, but:

- the `security` block is **reserved** (planned for a future release) and is **not populated**;
- `ipv4Configuration` is **reported read-only** — the reader's addressing is DHCP-assigned, and you **cannot set a static IP** through this API;
- **no IPv6** addressing is returned.
:::

:::note
**Disabled interface.** When Ethernet is disabled, the response contains only `ethConfig.interfaceDetails.interfaceName` and `status` (`Disabled`) — `linkStatus`, `ipv4Configuration`, `hostname`, and `macAddress` are absent. This is normal behavior and a `code 0` success, not an error.
:::

:::warning
**`apiVersion` is per-handler, not device-wide.** `get_eth` reports `apiVersion` `V1.1` (within its documented enum `V1.0` / `V1.1` / `V1.21`), but on the same firmware other commands (e.g. [`get_version`](/reference/mgmt/get-version), [`get_status`](/reference/mgmt/get-status)) report `V1.21`. The value is emitted by each command's handler, so do not treat it as a single device-wide constant.
:::

:::
