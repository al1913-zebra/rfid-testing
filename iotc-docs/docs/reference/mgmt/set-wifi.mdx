> 📕 **REFERENCE** · **Audience:** API consumer · **Use:** create or modify a Wi-Fi profile on the reader

## Overview

`set_wifi` writes a Wi-Fi profile to the reader — it **creates** a new saved profile or **modifies** an existing one (selected by `wifiConfig.operation` = `CREATE` / `MODIFY`). One payload configures the interface, the access point (ESSID, preference), the security type and its credentials (WPA2/WPA3 Personal or Enterprise), and DHCP IPv4. It is a state-changing command on the device-management (MGMT) endpoint. The response is status-only — it echoes **no configuration and no credentials**; confirm the result with [`get_wifi`](/reference/mgmt/get-wifi).

:::danger
`set_wifi` is **state-changing**. `accessPoint.connect: true` makes the reader **switch networks** — it disconnects the current profile and connects to the new one, which can **sever the MQTT control session** if the reader manages itself over Wi-Fi (see the notes below). Use `connect: false` to save a profile without switching. A profile you create is reversible with [`delete_wifi_profile`](/reference/mgmt/delete-wifi-profile).
:::

## When to Use

Use `set_wifi` to **provision or update the reader's Wi-Fi** — during onboarding, when a network changes, or to mark a profile preferred.

- **`CREATE`:** first-time provisioning of a new ESSID. Fails with code `18` if the ESSID already exists.
- **`MODIFY`:** update an existing profile (password, security, preference), matched by `essid`. Fails with code `15` if the ESSID is not found. A `MODIFY` can be a partial payload — e.g. just `operation` + `accessPoint.{isPreferred, essid}` to set the preferred flag.
- **Pre-provision safely:** send `connect: false` to save a profile without switching the active connection — the recommended pattern when managing a reader over the same Wi-Fi that carries the MDM session.

## Key Fields

- **`wifiConfig.operation`** — `CREATE` or `MODIFY`; determines whether a new profile is added or an existing one (matched by `essid`) is updated.
- **`wifiConfig.accessPoint.essid`** — the network name; the create target / modify key (case-sensitive).
- **`wifiConfig.accessPoint.connect`** — ⚠️ `true` switches the active connection (can sever the MDM session); `false` just saves. See the notes below.
- **`wifiConfig.accessPoint.security.securityType`** — selects which `securityDetails` key you populate (WPA2/WPA3 Personal or Enterprise).
- **`response.code`** — read this to determine success vs failure (see [Response Codes](#response-codes)).

## Response Codes

`set_wifi` returns `response.code` per `error_codes.json`. The codes applicable to this command are below; for the full table see the [Error codes reference](/reference/errors/codes) and [Error format](/reference/errors/format).

| Code | Description |
| --- | --- |
| `0` | Success — the profile was created or modified. |
| `2` | Invalid payload (`IOT_ERROR_INVALID_PAYLOAD`) — the JSON is malformed or has invalid field values. Validate against the schema and correct. |
| `15` | WIFI Error – SSID not found (`IOT_ERROR_SSID_NOT_FOUND`) — a `MODIFY` targeted an `essid` that is not saved. Verify the name or `CREATE` it first. |
| `17` | WIFI Error – SSID missed (`IOT_ERROR_SSID_MISSED`) — the required `essid` field is missing from the payload. |
| `18` | WIFI Error – SSID already exists (`IOT_ERROR_SSID_ALREADY_EXIST`) — a `CREATE` used an `essid` that already exists. Use `MODIFY`, or delete the existing profile first. |
| `19` | WIFI Error – SSID count overflow (`IOT_ERROR_SSID_LIMIT_OVERFLOW`) — the saved-profile limit is reached. Delete an unused profile with `delete_wifi_profile`. |
| `20` | Wifi is not supported (`IOT_ERROR_WIFI_INTER_NOT_SUPPORTED`) — the device has no Wi-Fi interface; use Ethernet. |
| `23` | Invalid enum value (`IOT_ERROR_INVALID_ENUM`) — a field (e.g. `operation`, `securityType`, `authentication`) is outside its allowed set. |
| `101` | Default error — generic catch-all. |

## Notes & Caveats

:::danger
**`connect: true` can sever the control session.** `accessPoint.connect: true` disconnects the currently connected profile and connects to the target ESSID. If the reader manages itself over the same Wi-Fi that carries its MQTT/MDM session, switching to a different network **drops the control channel** and can strand the device. When managing a reader over Wi-Fi, use `connect: false` (save without switching), and undo an unwanted profile with [`delete_wifi_profile`](/reference/mgmt/delete-wifi-profile) rather than connecting onto it.
:::

:::warning
**Credentials are write-only.** Supply passwords, passphrases, and identities in the request (masked here as `********`); the device never echoes them — `set_wifi`'s response carries no credentials, and `get_wifi` returns stored secrets masked.
:::

:::note
**`CREATE` vs `MODIFY`.** `CREATE` adds a new profile (fails `18` if the `essid` exists); `MODIFY` updates an existing profile matched by `essid` (fails `15` if not found). The device enforces a maximum number of saved profiles — exceeding it returns `19`; delete unused profiles with `delete_wifi_profile`. For Enterprise TLS, the `ca_cert` / `client_cert` / `client_key` referenced in `certificate` must already be installed via [`install_certificate`](/reference/mgmt/install-certificate).
:::

:::info
**IPv4 only, DHCP only.** In the current API version, `set_wifi` supports **IPv4 with DHCP assignment only**. The `ipv4Configuration` static fields (`ipAddress` / `subnetMask` / `gateway` / `dnsServer`) are modeled in the schema but **not supported yet** (planned for a future release), and **IPv6 is not available**. Use `enableDhcp: true`.
:::

:::note
**`apiVersion` is per-handler, not device-wide.** `set_wifi` reports `apiVersion` `V1.1` (within its documented enum `V1.0` / `V1.1`), but on the same firmware other commands (e.g. [`get_version`](/reference/mgmt/get-version)) report `V1.21`. Do not treat it as a single device-wide constant.
:::

:::tip
Because the response is status-only, **verify the result with [`get_wifi`](/reference/mgmt/get-wifi)** — check that the profile was saved (and its `isPreferred` / `accessPoint.status`). For the concept-level view, see [Wi-Fi networking](/infrastructure/network/wifi).
:::
